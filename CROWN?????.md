## 1. 这是什么

C.R.O.W.N 是一个运行在你电脑上的AI聊天程序，通过QQ和你聊天。

它和普通聊天机器人的区别：
- **有记忆**：它会记住你说过的话，以后聊天会自然提起
- **有性格**：它有自己的人设、说话风格、情绪
- **会成长**：聊得越多越了解你，回复越来越贴心
- **会主动找你**：不只等你来找它，它也会主动发消息
- **支持语音**：可以用语音回复你（需要配置TTS）
- **能接入QQ**：通过 NapCat 登录QQ、接收消息、发送回复
- **有安全护栏**：危机和边界风险会进入更稳妥的回复策略

---

## 2. 需要准备什么

1. **一台Windows电脑**（Win10/Win11）
2. **能上网**
3. **一个QQ号**（让AI用这个QQ和你聊天）
4. **10-15分钟耐心**

不需要懂编程。

## 3. 安装步骤

### 第1步：解压文件夹

把下载的压缩包解压到任意位置，比如桌面。

### 第2步：安装 Python

1. 打开 Python 官网下载页：https://www.python.org/downloads/windows/
2. 下载 Python 3.11 或 Python 3.12 的 Windows 安装包。
3. 双击安装包。
4. 第一屏一定要勾选 `Add python.exe to PATH`。
5. 点击 `Install Now`，等它安装完成。
6. 回到 CROWN 文件夹，在空白处按住 `Shift` 并右键，选择“在终端中打开”或“在 PowerShell 中打开”。
7. 输入：

```bat
python --version
```

能看到版本号，就说明 Python 安装好了。

### 第3步：创建 CROWN 专用环境

继续在刚才打开的终端里输入：

```bat
python -m venv venv
```

成功后，文件夹里会多出一个 `venv` 文件夹。它是 CROWN 自己专用的 Python 环境。

### 第4步：安装依赖

继续输入：

```bat
venv\Scripts\python.exe -m pip install --upgrade pip
venv\Scripts\python.exe -m pip install -r requirements.txt
```

如果下载很慢，改用下面这条：

```bat
venv\Scripts\python.exe -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 第5步：部署 NapCat

NapCat 是 CROWN 连接 QQ 的工具。没有它，机器人不能通过 QQ 收发消息。

1. 打开 NapCat 下载页：https://github.com/NapNeko/NapCatQQ/releases
2. 下载 Windows x64 版本压缩包，通常名字类似 `NapCat.Win.x64.zip`。
3. 在 CROWN 项目根目录中新建文件夹，名字必须是 `NapCat`。
4. 把压缩包里的内容全部解压到 `NapCat` 文件夹。
5. 确认能看到这个文件：`NapCat\napcat.bat`。

### 第6步：配置API

双击 `启动CROWN配置终端.bat`，会打开一个网页界面。

这是你的控制台，所有配置都在这里改。

**最重要的一步：配置API密钥。** 详见下一节。

### 第7步：启动机器人

双击 `重启文明（启动聊天ai）.bat`。

会弹出两个窗口，第一次需要扫码登录QQ。

完整机器人启动脚本不会自动弹出WebUI。需要修改配置时，单独双击 `启动CROWN配置终端.bat`。

### 第8步：开始聊天

用另一个QQ号给机器人QQ发消息就行了。

## 4. API配置详解

### 什么是API

API是AI模型的接口。你把消息发给API，API让AI思考，然后把回复传回来。

没有API，AI就没有"大脑"。

### 获取API密钥的步骤

#### 方案一：小米MiMo（推荐，国内可用）

1. 打开 https://xiaomimimo.com
2. 用小米账号登录
3. 进入控制台
4. 点击"API密钥"
5. 点击"创建新密钥"
6. 复制生成的密钥（类似 `sk-xxxxxxx`）

#### 方案二：通义千问（阿里，国内可用）

1. 打开 https://dashscope.console.aliyun.com
2. 用阿里云账号登录
3. 左侧菜单找到"API-KEY管理"
4. 点击"创建新的API-KEY"
5. 复制密钥

#### 方案三：DeepSeek（国内可用）

1. 打开 https://platform.deepseek.com
2. 注册账号
3. 左侧找到"API Keys"
4. 点击"创建API Key"
5. 复制密钥

#### 方案四：其他平台

配置终端内置了13个平台的预设，包括：
- 智谱AI（GLM）
- 月之暗面（Kimi）
- 零一万物
- 百度千帆
- 讯飞星火
- OpenAI（需要海外网络）
- Claude（需要海外网络）
- 任何OpenAI兼容API

步骤都是：注册 → 登录 → 找到API密钥页面 → 创建 → 复制。

### 在配置终端中填入密钥

1. 打开配置终端（双击 `启动CROWN配置终端.bat`）
2. 点击左侧"模型配置"
3. 找到"API平台"下拉框，选择你用的平台
   - 选择后API地址会自动填充
4. 在"API密钥"框里粘贴你的密钥
5. 选择模型（如 `mimo-v2.5-pro`）
6. 点击"保存配置"

### 配置TTS语音（可选）

如果你想让AI用语音回复你：

1. 在"模型配置"页面找到TTS设置
2. 填入TTS的API地址和密钥（通常和聊天API同一个平台）
3. 准备一个参考音频文件（角色的声音样本，wav格式）
4. 把音频文件放到 `voice/` 文件夹
5. 在配置中设置参考音频路径（默认 `./voice/Theresa.wav`）
6. 保存

### 配置搜索（可选）

如果想让AI能联网搜索：

1. 注册 Tavily（https://tavily.com）
2. 获取API Key
3. 在配置终端的"模型配置"中填入搜索API密钥
4. 保存

### config.yaml 文件说明

所有配置最终保存在 `config.yaml` 中，你也可以直接编辑这个文件：

```yaml
llm:
  api_base: "你的API地址/v1"    # 必须以/v1结尾
  api_key: "sk-xxxxxxx"         # 你的密钥
  model: "mimo-v2.5-pro"        # 模型名称
  light_model: "mimo-v2.5"      # 轻量模型（低负载时用）
  max_tokens: 2048              # 单次回复最大长度
  temperature: 0.75             # 创造性(0-1)，越高越随机
  timeout: 60.0                 # 超时秒数

tts:
  enabled: true
  api_base: "TTS的API地址"
  api_key: "***"
  model: "tts模型名称"
  reference_audio: "./voice/Theresa.wav"

search:
  api_key: "YOUR_TAVILY_KEY"
  max_results: 3

default_persona: "Theresa"      # 使用的人设
```

## 5. 启动和登录

### 启动机器人

双击 `重启文明（启动聊天ai）.bat`。

会弹出两个窗口：
- **机器人主程序**：显示运行日志
- **NapCat**：QQ协议端，负责和QQ通信

### NapCat 是什么

NapCat 是 CROWN 和 QQ 之间的连接层。它负责扫码登录机器人QQ号、接收QQ私聊消息、把消息交给 NoneBot/CROWN，再把 CROWN 生成的回复发回QQ。

可以把三者理解成：

| 部分 | 作用 |
|---|---|
| CROWN | 负责人设、记忆、心理画像、回复策略、主动消息 |
| NoneBot | 负责机器人事件框架，把QQ事件转成代码可处理的事件 |
| NapCat | 负责真正连接QQ、扫码登录、收发消息 |

如果 NapCat 没有启动或没有登录，CROWN 就不能通过QQ聊天。

### 首次登录

第一次启动需要扫码登录QQ：

1. NapCat窗口会显示一个二维码
2. 用手机QQ扫描这个二维码
3. 在手机上确认登录
4. 窗口显示"启动成功"或"已连接"

> NapCat下载地址（如需单独安装）：
> https://github.com/NapNeko/NapCatQQ/releases

### 登录后

登录成功后，用**另一个QQ号**给机器人QQ发消息，就能开始聊天了。

## 6. 开始聊天

### 基本聊天

直接发消息就行，像和朋友聊天一样。

可以做的事：
- 说"你好"打个招呼
- 分享今天发生的事
- 问问题（AI会联网搜索）
- 发图片（AI能看懂）
- 发表情包（AI能理解）
- 倾诉心事（AI会倾听和安慰）

### 常用命令

| 命令 | 功能 |
|------|------|
| `/clear` | 清空聊天记录，重新开始 |
| `/memory` | 查看AI记住了什么 |
| `/profile` | 查看关系状态和好感度 |
| `/debug` | 查看调试信息 |
| `/forget 关键词` | 让AI忘记某些记忆 |
| `/恋人模式 开/关` | 切换恋人模式 |
| `/relationship` | 查看/切换关系类型 |

### 小贴士

- 连续发多条消息，AI会合并成一条回复（7秒内）
- 深夜聊天AI会更温柔
- 聊得越多AI越了解你

## 11. 主动消息

AI会主动找你聊天，不会一直等你。

如果系统检测到危机场景或边界风险，普通主动消息会暂缓，避免在不合适的时候打扰。

在配置终端的"主动消息"页面可以设置：
- 主动消息间隔（默认0.5小时）
- 深夜静默时段（默认0-7点）
- 碎碎念开关
- 主动关心概率

## 12. 记忆系统

AI有多层记忆：

| 层级 | 说明 |
|------|------|
| 工作记忆 | 最近的对话 |
| 短期记忆 | 最近记住的事 |
| 长期记忆 | 重要的回忆（会衰减） |
| 深层记忆 | 模糊的印象 |
| 统一记忆账本 | 重要事实、偏好、目标、关系和安全记录 |

查看：`/memory`
遗忘：`/forget 关键词`

### 统一记忆账本

v0.0.6 新增 `MemoryLedger`，用来管理更重要、需要可追踪的记忆。

每条账本记忆包含：

- 类型：事实、事件、偏好、目标、风险、关系、观点、流程
- 来源：聊天、导入、WebUI、AI推断或工具
- 置信度：这条记忆有多可靠
- 敏感级别：low / medium / high / critical
- 确认状态：auto / confirmed / pending / rejected
- 证据：来自哪句话或哪段上下文

敏感内容默认不会直接当成确定事实长期使用，而是进入待确认状态。

在 WebUI 左侧的"记忆账本"页面里，你可以：

- 按用户、人设、类型、状态和关键词查找记忆
- 把待确认记忆改成已确认
- 把不确定记忆挂起
- 用新记忆覆盖旧记忆
- 删除错误记忆

覆盖不是简单删旧内容。CROWN 会给新记忆写入 `supersedes`，旧记忆变成 rejected，后续聊天优先使用新版记忆，减少“刚说过又忘”或新旧事实混在一起的问题。

检索时，CROWN 会综合考虑关键词相关度、记忆置信度、创建时间和确认状态。没有可靠记忆时，不应该硬编自己记得。

### 心理画像可解释化

心理画像现在会记录最近一次分析的证据、来源、置信度和更新时间。这样你能知道它为什么得出某个判断。

在 WebUI 的"心理健康与分析"页面里，还可以看到：

- 画像判断依据
- 置信度
- 来源
- 最近的心理画像变化历史

### 成长目标系统

成长目标系统用来记录你正在推进的事，比如学习、创作、生活习惯、情绪照顾、关系维护、工作和健康目标。

每个目标包含：

- 目标标题
- 目标类型
- 微任务
- 下次跟进时间
- 压力等级
- 是否允许主动提醒
- 当前状态

当你在聊天里明确说“我计划……”“我的目标是……”“我准备……”“我想完成……”时，CROWN 会自动创建一个候选目标。她之后能在聊天上下文里看见这些目标，但不会机械催促。

在 WebUI 左侧的"成长目标"页面里，你可以：

- 新建目标
- 搜索目标
- 编辑微任务和下次跟进时间
- 暂停目标
- 标记完成
- 删除错误目标

主动消息会读取到期目标跟进提示，但最终仍由 AI 结合时间、关系、安全状态和上下文判断现在该不该提。

### 主动陪伴调度

主动陪伴现在由 `ProactiveScheduler` 统一调度。它会把不同来源的主动消息放到同一套判断里：

- 定时任务提醒
- 成长目标跟进
- 主动关心事件
- 碎碎念
- 普通主动消息

每次调度都会写入 `proactive_events`，记录触发类型、原因、状态和时间。这样你能在 WebUI 的"主动消息"页面里看到：

- 为什么这次发了主动消息
- 为什么这次没有发
- 是目标跟进、定时任务、关心事件还是普通主动

调度器不会机械催促。它只是提供上下文和触发原因，最后仍由 AI 根据当前时间、关系状态、安全前置检查和人设判断该做什么。

### 自用版边界

CROWN自用版不安装、不启用“过度依赖”监控模块。SafetyMonitor只处理危机等级、边界风险、输出安全和主动消息安全前置检查。

## 14. 常见问题

**Q: 终端提示 python 不是内部或外部命令？**
A: 重新安装 Python，并确认安装第一屏勾选了 `Add python.exe to PATH`。安装后重新打开终端再试。

**Q: 安装依赖很慢或失败？**
A: 检查网络，或使用清华源命令：`venv\Scripts\python.exe -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`。

**Q: AI不回复？**
A: 检查API密钥、网络、QQ登录状态、白名单。

**Q: 提示API错误？**
A: 密钥不对。检查有没有复制完整、有没有过期、余额够不够。

**Q: 回复很奇怪？**
A: 发 `/clear` 清空重来。

**Q: 怎么换性格？**
A: 配置终端 → 人设控制。

**Q: 怎么限制谁能聊天？**
A: 配置终端 → 白名单 → 添加QQ号。

**Q: 怎么更新版本？**
A: 下载新版，把 config.yaml、personas/、data/、voice/ 复制过去。首次启动自动迁移。

**Q: 数据库在哪？**

## 陪伴中枢怎么用

陪伴中枢是 WebUI 里的总览页，适合用来确认机器人现在参考了哪些信息，以及主动消息为什么发或不发。

1. 打开 `启动CROWN配置终端.bat`。
2. 浏览器进入 `http://127.0.0.1:5050`。
3. 在左侧点击「陪伴中枢」。
4. 顶部可以填写用户 ID、人设名称，也可以留空直接看总体摘要。
5. 页面会显示记忆账本、成长目标、主动消息事件、安全状态、心理画像、关系绑定和 AI 状态。
6. 想看详细内容时，点击卡片右上角的详情按钮进入对应页面。

这个页面只负责查看，不会直接修改记忆、目标、关系或聊天数据。CROWN 自用版不包含“过度依赖”监控模块。

## 系统自检怎么用

系统自检是 WebUI 里的诊断页，适合在更新后确认每个板块是否真的接到了文件、接口和数据库。

1. 打开 `启动CROWN配置终端.bat`。
2. 浏览器进入 `http://127.0.0.1:5050`。
3. 在左侧点击「系统自检」。
4. 点击「刷新自检」。
5. 先看「聊天与数据库模块」：这里会列出聊天入口、记忆账本、成长目标、主动消息、安全护栏、关系定制、账号绑定和心理画像。
6. 再看「WebUI 页面联通性」：这里会显示每个页面是否有 API，作用对象是文件、运行时状态还是数据库。
7. 最后看「数据库与表」：这里能确认当前项目实际发现了哪些数据库和表。

如果看到「等待数据」，通常不是坏掉，而是相关功能还没有在聊天中触发过，所以数据库表还没有产生。真正需要优先修的是「接口缺失」和「文件缺失」。

## 她回复前会参考什么

聊天主流程在 `src/core/pipeline.py`，回复前上下文由 `src/core/context_builder.py` 统一拼装。

你在 WebUI 里调整的这些内容，会通过上下文构建层进入聊天：

- 关系定制和账号绑定。
- 记忆账本、长期记忆、情景记忆。
- 成长目标和到期跟进。
- 心理画像、情绪状态和人格控制。
- 当前时间、生活状态、场景识别。

所以如果你想确认某个模块有没有真的影响聊天，可以先去「系统自检」看文件/API/表是否可用，再去「陪伴中枢」看当前读取到的具体状态。

## 安全护栏怎么接入聊天

聊天安全入口在 `src/core/safety_coordinator.py`。

它会在用户消息进入聊天主流程时先做风险评估。如果判断需要直接安全回复，就不会继续让普通聊天模型自由发挥；如果不需要直回，就把必要的安全协议加入上下文。模型生成回复后，还会再经过一次输出护栏。

主动消息也会先经过安全前置检查，避免在危机或边界风险状态下贸然打扰。

CROWN 自用版不包含“过度依赖”监控模块。

## 记忆写入怎么接入聊天

聊天记忆写入入口在 `src/core/memory_coordinator.py`。

用户说的话会先进入长期记忆抽取和 MemoryLedger 判断。AI 回完后，回复会写入短期记忆；聊天达到一定轮数后，会触发长期记忆 AI 抽取；如果这轮对话属于秘密、情感共鸣、关心回应或明显情绪事件，还会写入情景记忆。

如果你发现她刚听过的话转头又问，可以这样查：

1. 打开 WebUI 的「系统自检」，看记忆模块是否可用。
2. 打开「数据库」，检查 `chat_history` 是否有最近对话。
3. 打开「记忆账本」，检查重要事实是否进入 `memory_ledger`。
4. 查看 `debug.log` 里的 `[memory]` 和 `[memory-ledger]` 日志。

## 成长目标和关系成长怎么接入聊天

成长状态入口在 `src/core/growth_coordinator.py`。

每次正常回复或流式回复完成后，它会更新关系成长统计。如果你的聊天里明确出现计划、目标、打算、想完成的内容，它会尝试创建成长目标；聊天轮数达到条件时，还会触发成长总结。它也会把这轮对话记录成日常分享、分享秘密、关心回应、情感共鸣或日常问候。

这些数据会进入成长目标页、陪伴中枢和后续回复上下文。想确认是否生效，可以这样查：

1. 打开 WebUI 的「系统自检」，看成长目标模块是否可用。
2. 打开「成长目标」，看聊天里提到的计划是否进入目标列表。
3. 打开「陪伴中枢」，看当前用户的成长目标、关系状态和最近事件。
4. 查看 `debug.log` 里的 `[growth-goal]` 日志。

## 工具调用怎么接入聊天

工具路由入口在 `src/core/tool_router.py`。

聊天回复前，工具路由协调器会把工具能力提示注入上下文；当模型回复里出现工具调用信号时，它会执行工具，把工具结果回注给模型，再生成二次回复。二次回复仍会经过安全护栏，所以不会绕开原有安全边界。

你可以这样确认是否生效：

1. 打开 WebUI 的「系统自检」，看聊天入口模块是否包含 `src/core/tool_router.py`。
2. 在聊天中触发一个工具型请求，观察回复是否出现基于工具结果的二次回答。
3. 查看 `debug.log` 里的工具调用日志，确认触发与回注链路完整。

## 人设稳定评审怎么接入聊天

评审入口在 `src/core/critic_coordinator.py`。

它会在回复前尝试注入人格漂移修正提示；回复后会缓存文本、异步触发漂移检测，并对这条回复做残差与可采纳性判定。这样可以把“是否偏离人设”从主回复链路里独立出来，方便后续定位问题。

你可以这样确认是否生效：

1. 打开 WebUI 的「系统自检」，看聊天入口模块是否包含 `src/core/critic_coordinator.py`。
2. 连续聊几轮后查看 `debug.log`，确认出现 persona 相关评审日志。
3. 如果出现异常口癖或人设偏移，再结合陪伴中枢和系统自检定位上下文与评审链路。

## 聊天记录导入（WebUI）

这个页面用于把历史聊天导入到系统里，作为后续关系简史、记忆冷启动、心理画像辅助分析的基础数据。

操作步骤：
1. 打开 `启动CROWN配置终端.bat`。
2. 进入左侧「聊天记录导入」。
3. 输入 `user_id`（必填），可选填 `persona`。
4. 选择以下任一方式导入：
   - 粘贴聊天记录文本（支持普通回车分行）
   - 上传聊天记录文件（zip/html/htm/txt/mht）
5. 查看导入统计、最近分析和最近消息。

说明：
- 导入页面会写入 `chat_records` 和 `chat_record_analysis`。
- 它不会直接修改记忆账本、关系状态、成长目标。
- 自用版 CROWN 不包含“过度依赖”监控，本页面也不会接入该模块。

## 关系简史（WebUI）

这个模块在「关系定制」页面里，作用是把与关系判断相关的事件放在一条时间线上，方便你快速看明白“她现在为什么这么回”。

操作步骤：
1. 打开 `启动CROWN配置终端.bat`。
2. 进入左侧「关系定制」。
3. 在“关系简史（只读）”输入 `user_id`（建议填写），可选填写 `persona`。
4. 点击“刷新简史”。
5. 先看摘要（当前关系、最近切换、绑定关系、最近联系），再看时间线（关系切换、绑定、目标事件、记忆、聊天、长期记忆）。

说明：
- 关系简史只负责读取和展示，不会直接改写聊天或记忆数据。
- 如果某些表还没在该账号下生成，会看到空态或部分事件，这是正常兼容行为。
- CROWN 自用版不包含“过度依赖”监控模块。

## 长期记忆冷启动摘要（WebUI）

这个模块在「聊天记录导入」页面，用来把你导入的历史聊天快速整理成“可追踪的初始记忆”，避免机器人一开始没有上下文。

操作步骤：
1. 先在「聊天记录导入」完成文本或文件导入。
2. 保持同一个 `user_id`（可选 `persona`）。
3. 点击“生成冷启动摘要”。
4. 查看“冷启动摘要结果”中的候选内容、关键词和入账数量。

说明：
- 摘要会写入 `memory_ledger` 与 `long_term_memory`，后续可在记忆账本页确认或调整。
- 没有导入聊天记录时会返回空态，不会报 500。
- CROWN 自用版不包含“过度依赖”监控模块。

## 评测控制台（WebUI）

这个模块在左侧「评测控制台」，用于快速做一次“链路健康检查”。

操作步骤：
1. 打开 `启动CROWN配置终端.bat`。
2. 进入「评测控制台」。
3. 可选填写 `user_id`、`persona`。
4. 勾选要评测的场景后点击“运行评测”。
5. 在“历史报告”里按状态和 `report_id` 关键词筛选查看。

说明：
- 评测控制台是只读评估页，不会直接改写聊天数据或配置。
- 若出现 `waiting_db` 或空态，通常是该账号尚未触发对应能力，不代表代码故障。
- CROWN 自用版不包含“过度依赖”监控模块。

## 高级配置自动归类（WebUI）

为了减少导航拥挤，WebUI 会自动把复杂模块归到「高级设置」分组（如系统自检、评测控制台、记忆账本、聊天记录导入、数据库等）。

说明：
- 这只影响页面分组展示，不改变模块本身功能。
- 你平时优先使用常用配置区，排查问题或深度调试时再进入高级设置即可。
