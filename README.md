# C.R.O.W.N v0.0.6

## 快速开始

1. 运行 `一键部署.bat` 安装环境
2. 运行 `重启文明（启动聊天ai）.bat` 启动机器人
3. 首次启动需要扫码登录 QQ（NapCat 窗口会显示二维码）
4. 如需配置，单独运行 `启动CROWN配置终端.bat` 打开 WebUI（端口 5050）

**NapCat 请解压到 CROWN 根目录下的 `NapCat` 文件夹中。**

## NapCat 是做什么的

NapCat 是 QQ 协议端。CROWN 本身负责人设、记忆、心理画像、安全护栏、主动消息和调用大模型；NapCat 负责登录 QQ、接收 QQ 消息、把消息转给 NoneBot，再把 CROWN 生成的回复发回 QQ。

简单说：

- **CROWN**：负责“她怎么想、怎么记、怎么回复”。
- **NoneBot**：负责机器人事件框架。
- **NapCat**：负责真正连接 QQ 和收发消息。

没有 NapCat，CROWN 仍能作为代码项目存在，但不能通过 QQ 正常收发私聊消息。

## 陪伴中枢

陪伴中枢是 WebUI 的只读总览模块，用来统一查看她当前可参考的状态：记忆账本、成长目标、主动消息判断、安全状态、心理画像、关系绑定和 AI 配置。

使用方式：
1. 运行 `启动CROWN配置终端.bat` 打开 WebUI。
2. 在左侧导航进入「陪伴中枢」。
3. 可填写用户 ID、人设名称并选择最近记录数量；留空时显示默认人设和全部可读数据的摘要。
4. 查看「主动消息判断」可知道最近为什么发送、跳过或失败。
5. 点击各卡片右上角按钮可跳到对应详情页，例如记忆账本、成长目标、主动消息事件、关系页或心理健康分析页。

注意事项：
- 陪伴中枢只读，不会直接修改聊天数据、记忆、目标或关系配置。
- CROWN 自用版不安装、不启用“过度依赖”监控模块；陪伴中枢只展示危机等级、边界风险、输出安全、主动消息前置安全检查等必要状态。

## 系统自检

系统自检是 WebUI 的只读诊断模块，用来整理项目结构并确认各板块是否真正接到文件、接口和数据库。

它会检查：

- 聊天入口、人设 Prompt、记忆账本、成长目标、主动消息、安全护栏、关系定制、账号绑定和心理画像等模块文件是否存在。
- WebUI 每个页面是否有对应 API。
- 页面是否连接到真实文件、运行时状态或数据库表。
- 当前已发现的数据库文件、表名和表数量。

使用方式：
1. 运行 `启动CROWN配置终端.bat` 打开 WebUI。
2. 在左侧导航进入「系统自检」。
3. 点击「刷新自检」查看模块、页面和数据库状态。
4. 状态为「可用」表示文件/API/数据库连接已接上；「等待数据」表示代码和接口存在，但当前数据库还没有触发生成对应表。
5. 可从自检页跳转到「数据库」或「陪伴中枢」继续检查具体记录。

系统自检只读，不会写入或删除聊天记录、记忆、目标、关系或配置。

## 内部上下文构建

聊天主流程由 `src/core/pipeline.py` 负责调度，`src/core/context_builder.py` 负责集中拼装她回复前需要参考的上下文。

上下文构建会读取并组织：

- 当前时间、生活状态和场景识别。
- 长期记忆、情景记忆和记忆账本。
- 情绪状态、PAD 状态、心理画像和人格控制状态。
- 关系定制、账号关系绑定和成长目标。
- 内容策略、用户数据摘要和额外上下文。

这样 WebUI 中修改的关系、账号绑定、成长目标、记忆和心理画像会通过上下文构建层进入聊天提示词，真实影响回复，而不是只停留在配置页面。

## 安全协调器

安全协调器位于 `src/core/safety_coordinator.py`，负责把聊天主流程里的安全动作集中起来。

它会处理：

- 接收消息后的危机等级和边界风险评估。
- 高危输入的直接安全回复。
- 回复前的安全协议提示注入。
- 回复后的输出护栏。
- 主动消息发送前的安全前置检查。

底层仍使用 `src/safety/safety_monitor.py` 的危机/边界能力。CROWN 自用版不安装、不启用“过度依赖”监控模块。

## 记忆协调器

记忆协调器位于 `src/core/memory_coordinator.py`，负责把聊天主流程里的记忆写入动作集中起来。

它会处理：

- 用户输入进入长期记忆抽取。
- 明确事实、偏好、目标和风险进入 MemoryLedger。
- AI 回复写入短期记忆。
- 每隔一定轮数触发长期记忆 AI 抽取。
- 按事件类型和情绪强度写入情景记忆。

记忆读取仍由上下文构建层负责，记忆写入由记忆协调器负责，方便排查“刚说过又忘”的问题。

## 成长协调器

成长协调器位于 `src/core/growth_coordinator.py`，负责把聊天主流程里的成长状态更新集中起来。

它会处理：

- 聊天后更新关系成长统计。
- 从用户明确提到的计划、目标、打算中创建成长目标。
- 每隔一定轮数触发成长总结。
- 按聊天内容和情绪把对话记录为日常分享、分享秘密、关心回应、情感共鸣或日常问候。

成长协调器写入的数据会被成长目标页、陪伴中枢和上下文构建层读取。聊天里产生的目标和关系成长会进入数据库，并在后续回复、主动消息判断和 WebUI 总览中发挥作用。

## 工具路由协调器

工具路由协调器位于 `src/core/tool_router.py`，负责把聊天中的 MCP 工具路由动作集中起来。

它会处理：

- 在回复前注入工具能力提示（当工具可用时）。
- 检测模型回复里是否包含工具调用信号。
- 工具调用成功后，把工具结果回注到对话并生成二次回复。
- 二次回复同样走安全输出护栏，保持和主回复一致的安全边界。

这样 `pipeline.py` 只保留主流程调度，工具策略从主流程中剥离，后续做评测和工具守卫时更容易定位问题。

## 评审协调器

评审协调器位于 `src/core/critic_coordinator.py`，负责把回复后的“人设稳定性评审”动作集中起来。

它会处理：

- 回复前注入人格漂移修正提示（如果存在）。
- 回复后缓存文本用于人格漂移检测。
- 异步触发人格漂移检查任务。
- 计算残差标签并执行可采纳性判定，必要时生成修正信号。

这样人格稳定相关的评审链路从 `pipeline.py` 主流程中独立，后续做回归评测和人设稳定性排查时更清晰。

## 常见问题

**Q: 启动闪退？**
A: 右键 → 以管理员身份运行。确保 Python 已安装。

**Q: NapCat 不启动？**
A: 确保 `NapCat` 文件夹在 CROWN 根目录下，里面有 `napcat.bat`。

**Q: 机器人不回消息？**
A: 检查 NapCat 窗口是否显示"已连接"。如果没有，重新扫码登录。

**Q: WebUI 打不开？**
A: 确认端口 5050 没被占用。运行 `启动CROWN配置终端.bat` 单独启动 WebUI。

## 聊天记录导入

聊天记录导入是 WebUI 的导入与总览模块，用来把历史聊天（文本粘贴或文件）写入 `chat_records` 与 `chat_record_analysis`，为关系简史、记忆冷启动和后续分析提供底座数据。

使用方式：
1. 打开 `启动CROWN配置终端.bat` 进入 WebUI。
2. 左侧进入「聊天记录导入」。
3. 填写 `user_id`（必填），可选填写 `persona`。
4. 选择「粘贴聊天记录」或「上传聊天记录文件（zip/html/txt/mht）」。
5. 在同页查看导入统计、最近分析和最近消息。

注意事项：
- 该页会写入聊天导入表，但不会直接改写记忆账本、关系状态或成长目标。
- `user_id` 为空时返回空态，不会触发 500 报错。
- CROWN 自用版不安装、不启用“过度依赖”监控模块，本模块也不接入该能力。

## 关系简史

关系简史是 WebUI「关系定制」页里的只读总览卡片，用来把关系切换、账号绑定、成长事件、关系相关记忆、聊天记录与长期记忆放在同一条时间线里，方便判断“她为什么会用当前关系和语气回复”。
使用方式：
1. 打开 `启动CROWN配置终端.bat`。
2. 进入「关系定制」页面。
3. 在“关系简史（只读）”输入 `user_id`（建议填写）和可选 `persona`，然后点击“刷新简史”。
4. 先看摘要区（当前关系、最近切换、绑定关系、最近联系），再看下方时间线事件。
注意事项：
- 关系简史只读，不会直接修改聊天数据。
- 如果某些数据表暂未生成，会展示空态或部分事件，不会导致 WebUI 卡死。
- CROWN 自用版不安装、不启用“过度依赖”监控模块。

## 长期记忆冷启动摘要

长期记忆冷启动摘要位于「聊天记录导入」页，用于把已导入聊天自动提炼为可追踪的初始记忆候选，并同步入账到 `memory_ledger` 与 `long_term_memory`，让新账号也能快速建立连续上下文。
使用方式：
1. 在「聊天记录导入」先导入该用户聊天记录。
2. 确认 `user_id`（可选 `persona`）后点击「生成冷启动摘要」。
3. 在“冷启动摘要结果”查看候选内容、关键词与入账数量。
注意事项：
- 该功能基于已导入聊天记录做规则提炼，不直接伪造事实。
- 冷启动结果会写入数据库，请在记忆账本页继续确认与整理。
- CROWN 自用版不安装、不启用“过度依赖”监控模块。

## 评测控制台

评测控制台位于 WebUI 左侧「评测控制台」，用于把聊天链路的关键能力做一次可复用的只读评估，快速判断“当前是没数据、还是链路故障”。

使用方式：
1. 打开 `启动CROWN配置终端.bat`。
2. 进入「评测控制台」。
3. 可选填写 `user_id`、`persona`，勾选要运行的场景。
4. 点击「运行评测」，查看本次分数、通过/警告/失败明细。
5. 在历史报告区域按状态和 `report_id` 关键词筛选，点「查看」回放某次报告。

注意事项：
- 评测控制台是只读评估页，不直接修改聊天数据和配置。
- `waiting_db` 或空态通常代表“该账号还没触发过这块能力”，不等于程序故障。
- CROWN 自用版不安装、不启用“过度依赖”监控模块。

## 高级配置自动归类

WebUI 现在会自动把复杂模块归入「高级设置」分组（例如系统自检、评测控制台、记忆账本、聊天记录导入、数据库等），避免普通配置区过载。

使用方式：
1. 正常打开 WebUI。
2. 在左侧先使用常用配置区（模型、人设、场景、语气、主动消息等）。
3. 需要深度排查或底层数据操作时，再进入「高级设置」分组。

注意事项：
- 自动归类只影响导航展示层，不会改变接口行为或数据库数据。
- 后续新增复杂页只要标记为复杂项，会自动进入高级设置分组。
