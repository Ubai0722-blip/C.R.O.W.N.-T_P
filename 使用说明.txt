══════════════════════════════════════════════════════════════
  一、快速开始（3步搞定）
══════════════════════════════════════════════════════════════

【第1步】一键部署
  找到文件夹里的 "一键部署.bat"，双击它。
  它会自动帮你做这些事：
    - 检查电脑上有没有 Python（没有就自动下载安装）
    - 安装所有需要的依赖包
  整个过程大概需要3-10分钟，耐心等一下。
  看到"部署完成！"就表示成功了。

  注意：如果双击闪退了，右键点它，选"以管理员身份运行"。

【第2步】网页配置
  双击 "启动CROWN配置终端.bat"，会打开一个网页界面。
  你需要做的最重要的事情：
    - 找到"模型配置"
    - 填入你的 API 密钥（API Key）
    - 选择你要用的模型
  其他设置可以先不改，以后慢慢调。

  API 密钥从哪里来？
    你需要一个支持 OpenAI 格式的 API 服务。
    常见的选择：小米MiMo、通义千问、DeepSeek 等。
    去对应的平台注册账号，创建 API Key，复制粘贴过来就行。
    详细配置方法见下方"三、API配置详解"。

【第3步】准备 NapCat
  NapCat 是 QQ 协议端，机器人通过它收发 QQ 消息。
  CROWN 负责人设、记忆、回复和安全护栏；NapCat 负责登录 QQ、接收消息、发送回复。
  NoneBot 在中间接收 NapCat 事件，再把消息交给 CROWN 处理。
  请将 NapCat 解压到 CROWN 根目录下的 "NapCat" 文件夹中。
  确保文件夹结构为：CROWN/NapCat/napcat.bat
  首次启动需要扫码登录 QQ。

【第4步】启动机器人
  双击 "重启文明（启动聊天ai）.bat"。
  会弹出两个窗口：
    - 一个是机器人主程序
    - 一个是QQ协议端（NapCat）
  第一次启动需要扫码登录QQ：
    - 用手机QQ扫描窗口里显示的二维码
    - 登录成功后，窗口会显示"启动成功"
  然后用另一个QQ号给机器人QQ发消息，就能开始聊天了！

  注意：完整机器人启动脚本不会自动弹出WebUI。
  如需修改配置，请另外双击 "启动CROWN配置终端.bat"。

  NapCat 下载地址（如果需要单独安装）：
  https://github.com/NapNeko/NapCatQQ/releases

【NapCat到底是什么】
  - 它不是AI大脑，也不负责人设。
  - 它是QQ连接层，负责扫码登录、维持QQ在线、收发QQ消息。
  - CROWN没有NapCat也能运行部分本地代码，但不能通过QQ和你聊天。
  - 如果机器人不回消息，优先看NapCat是否已登录、是否显示连接成功。


══════════════════════════════════════════════════════════════
  二、白名单功能（重要！）
══════════════════════════════════════════════════════════════

白名单功能可以控制哪些QQ号能和你的AI聊天。

【自动机制】
  第一个给机器人发消息的用户会自动加入白名单。
  所以你不用担心配置问题，先聊起来就行。

【管理白名单】
  方式一：网页界面管理（推荐）
    1. 打开"启动CROWN配置终端.bat"
    2. 点击左侧导航栏的"白名单"
    3. 在这里你可以：
       - 查看所有白名单用户
       - 启用/禁用某个用户
       - 手动添加新用户
       - 删除用户

  方式二：直接操作数据库
    白名单数据存在 data/personas/{人设}/global.db 的 chat_whitelist 表中。
    可以用配置终端的"数据库"页面查看和修改。

【白名单规则】
  - 只有"已启用"的用户才能和AI聊天
  - 不在白名单中的用户发消息会被直接忽略（不会有任何提示）
  - 如果白名单表是空的，第一个发消息的用户会自动添加
  - 出错时默认放行，避免把所有人锁死在外面


══════════════════════════════════════════════════════════════
  三、API配置详解
══════════════════════════════════════════════════════════════

C.R.O.W.N 需要一个支持 OpenAI 格式的 API 服务才能运行。
本节详细介绍如何获取和配置 API。

【什么是API？】
  API 就是"接口"，是你和AI模型之间的桥梁。
  你把消息发给API，API让AI模型思考，然后把回复传回来。
  没有API，AI就没有"大脑"，无法回复你。

【支持的API平台】
  配置终端内置了以下平台的预设，选择后自动填充API地址：

  1. 小米MiMo（推荐）
     地址：https://token-plan-cn.xiaomimimo.com/v1
     模型：mimo-v2.5-pro（主力）/ mimo-v2.5（轻量）
     注册：访问小米MiMo开放平台，用小米账号登录
     获取密钥：控制台 -> API密钥 -> 创建新密钥

  2. 通义千问（阿里）
     地址：https://dashscope.aliyuncs.com/compatible-mode/v1
     模型：qwen-plus / qwen-turbo
     注册：访问 dashscope.console.aliyun.com
     获取密钥：控制台 -> API-KEY管理

  3. DeepSeek
     地址：https://api.deepseek.com/v1
     模型：deepseek-chat
     注册：访问 platform.deepseek.com
     获取密钥：API Keys页面

  4. 智谱AI（GLM）
     地址：https://open.bigmodel.cn/api/paas/v4
     模型：glm-4-flash
     注册：访问 open.bigmodel.cn

  5. 月之暗面（Kimi）
     地址：https://api.moonshot.cn/v1
     模型：moonshot-v1-8k
     注册：访问 platform.moonshot.cn

  6. 零一万物
     地址：https://api.lingyiwanwu.com/v1
     模型：yi-large
     注册：访问 platform.lingyiwanwu.com

  7. 百度千帆
     地址：https://aip.baidubce.com/rpc/2.0/ai_custom/v1/wenxinworkshop
     注册：访问 qianfan.cloud.baidu.com

  8. 讯飞星火
     地址：https://spark-api-open.xf-yun.com/v1
     注册：访问 xinghuo.xfyun.cn

  9. OpenAI（需要海外网络）
     地址：https://api.openai.com/v1
     模型：gpt-4o / gpt-4o-mini
     注册：访问 platform.openai.com

  10. Claude（需要海外网络）
      地址：https://api.anthropic.com/v1
      注册：访问 console.anthropic.com

  11. 自定义OpenAI兼容API
      任何支持 OpenAI 格式的第三方 API 都可以用
      填入对应的 API地址 和 API密钥 即可

【配置步骤】
  1. 双击"启动CROWN配置终端.bat"打开网页界面
  2. 点击左侧"模型配置"
  3. 找到"API平台"下拉框，选择你使用的平台
     - 选择后API地址会自动填充
  4. 在"API密钥"输入框粘贴你的密钥
     - 注意：密钥不要有多余空格
     - 注意：密钥要完整复制，不要漏字符
  5. 在"模型名称"选择或输入你要用的模型
  6. 点击"保存配置"

  也可以同时配置TTS（语音合成）：
  7. 找到TTS设置区域
  8. 填入TTS API地址和密钥
  9. 设置参考音频路径（默认 ./voice/Theresa.wav）
  10. 保存

【配置文件说明】
  所有配置最终保存在 config.yaml 文件中：
  
  # LLM（聊天AI）配置
  llm:
    api_base: "你的API地址/v1"    # API地址，必须以/v1结尾
    api_key: "***"                # 你的API密钥
    model: "模型名称"              # 要使用的模型
    light_model: "轻量模型"        # 用于低负载场景的轻量模型
    max_tokens: 2048              # 单次回复最大长度
    temperature: 0.75             # 创造性(0-1)，越高越随机
    timeout: 60.0                 # 请求超时秒数

  # TTS（语音合成）配置
  tts:
    enabled: true                 # 是否启用语音
    api_base: "TTS的API地址"
    api_key: "***"
    model: "tts模型名称"
    reference_audio: "./voice/Theresa.wav"  # 音色参考音频

  # 搜索配置
  search:
    api_key: "tvly-***"          # Tavily搜索API密钥
    max_results: 3                # 搜索结果数量

  # 默认人设
  default_persona: "Theresa"      # 使用的人设名称

【常见API问题】
  Q: 提示"API错误"或"401"？
  A: 密钥不对。检查：
     - 有没有复制完整
     - 有没有多余空格
     - 密钥有没有过期
     - 余额够不够

  Q: 提示"超时"？
  A: 网络问题或模型太慢。试试：
     - 检查网络连接
     - 增大timeout值
     - 换一个更快的模型

  Q: 回复内容被截断？
  A: max_tokens太小。增大这个值即可。

  Q: AI回复很死板？
  A: temperature太低。建议设为0.7-0.9。


══════════════════════════════════════════════════════════════
  四、数据库管理
══════════════════════════════════════════════════════════════

v0.0.4版本对数据库进行了重大升级，采用"账户-人设"隔离结构。

【新版数据库结构】
  每个用户在每个人设下有独立的数据库文件：
    data/accounts/{QQ号}/{人设名}/user_data.db

  例如用户 qq_517908311 使用 Theresa 人设：
    data/accounts/qq_517908311/Theresa/user_data.db

  这意味着：
    - 不同用户的聊天记录完全隔离
    - 同一用户切换人设后数据独立
    - 每个用户的数据库包含该用户的所有数据

【统一记忆账本】
  新版本新增 memory_ledger 表，用来统一管理更重要的记忆：
    - fact：事实
    - event：事件
    - preference：偏好
    - goal：目标
    - risk：危机/边界相关安全记录
    - relationship：关系相关记忆

  每条账本记忆会记录：
    - 来源 source
    - 置信度 confidence
    - 敏感级别 sensitivity
    - 是否确认 consent_status
    - 版本覆盖 supersedes
    - 创建时间和最后使用时间

  高敏感内容默认进入 pending，后续可以在WebUI/API里确认、挂起、覆盖或删除。

  WebUI左侧"记忆账本"页面可以：
    - 按用户、人设、类型、状态和关键词搜索
    - 确认 pending 记忆，让它可用于聊天
    - 挂起不确定记忆，暂时不当成确定事实
    - 用新记忆覆盖旧记忆，建立 supersedes 链
    - 删除错误记忆，并同步清理 FTS 检索表

  聊天召回时会综合考虑：
    - FTS5/BM25 相关度
    - 记忆置信度
    - 创建时间权重
    - 确认状态

  如果账本没有可靠记忆，AI不应该编造“记得”的内容。

【自用版安全边界】
  CROWN自用版不安装、不启用“过度依赖”监控模块。
  SafetyMonitor只处理危机等级、边界风险、输出安全和主动消息安全前置检查。

【心理画像可解释化】
  在 WebUI 的"心理健康与分析"页面中，用户画像概览会显示：
    - 最近判断依据
    - 置信度
    - 来源
    - 心理画像变化历史

  这些信息用于解释画像判断从哪里来，方便检查和修正。

【成长目标系统】
  新版本新增 growth_goals 表，用来记录用户正在推进的目标：
    - title：目标标题
    - goal_type：学习/创作/生活习惯/情绪照顾/关系维护/工作/健康/其他
    - micro_tasks：微任务列表
    - next_follow_up：下次跟进时间
    - pressure_level：压力等级 0-5
    - allow_proactive：是否允许主动跟进
    - status：active/paused/completed/archived

  当用户在聊天里明确说“计划、目标、打算、准备、想要完成”等内容时，
  CROWN会自动创建一个候选目标，并把它写入成长目标系统。

  WebUI左侧"成长目标"页面可以：
    - 按用户、状态和关键词搜索目标
    - 新建目标
    - 编辑目标标题、微任务和下次跟进时间
    - 暂停或完成目标
    - 删除错误目标

  主动消息会读取到期目标，但不会机械催促。AI会结合当前时间、关系状态、
  安全前置检查和上下文，判断现在该不该提、怎么提。

【主动陪伴调度】
  新版本新增 ProactiveScheduler，用来统一判断主动消息来源：
    - scheduled_task：定时任务到期
    - goal_followup：成长目标到了跟进时间
    - care_event：存在可关心事件
    - mutter：碎碎念
    - ordinary_proactive：普通主动消息
    - skipped：本轮不适合主动

  每次判断都会写入 proactive_events 表：
    - trigger_type：触发类型
    - reason：触发或跳过原因
    - status：planned/sent/skipped/failed
    - extra_context：给 AI 的上下文
    - created_at/sent_at：判断和发送时间

  WebUI左侧"主动消息"页面可以查看最近调度事件，知道为什么发或为什么没发。

  该模块只处理时机、上下文、冷却、安全前置和触发原因记录，不包含“过度依赖”监控。

【人设全局数据】
  不属于特定用户的数据存在人设级全局库中：
    data/personas/{人设名}/global.db
  
  包含：生活事件、聊天白名单、关系状态

【共享心理画像】
  跨人设共享的心理分析数据：
    data/shared/psychology.db

【自动迁移】
  如果你有旧版的 data/ 目录（v0.0.3及之前），首次启动新版时
  程序会自动检测并迁移：
    - 旧版 chatbot_{人设}.db → 按用户拆分到 accounts/ 目录
    - 旧版 chatbot_shared.db → shared/psychology.db
    - 旧文件备份到 data/_legacy_backup/
    - 迁移过程全自动，无需手动操作
    - 迁移幂等安全，重复运行不会破坏数据

【WebUI数据库管理】
  打开配置终端，点击左侧"数据库"：

  - 用户筛选：顶部下拉框选择用户账号，按用户过滤显示数据
  - 表按钮：点击表名查看数据，悬停显示中文名和作用说明
  - 数据操作：每行有"查看/编辑/删除"按钮
  - 新增数据：点击"+ 新增数据"按钮
  - 自动刷新：支持3/5/10秒自动刷新间隔
  - 跨库合并：自动合并全局库和所有用户库的数据

──────────────────────────────────────────────────────────────
  6.1 这是什么东西？
──────────────────────────────────────────────────────────────

C.R.O.W.N 是一个可以24小时陪你聊天的AI程序。

它不是普通的聊天机器人，而是一个有"记忆"、有"性格"、
会"成长"的AI伴生系统。

简单来说：
  - 你跟它聊得越多，它就越了解你
  - 它会记住你说过的事情，以后聊天时会自然提起
  - 它有自己的情绪，会开心、会难过、会撒娇
  - 它会主动找你聊天（碎碎念），不会一直等你来找它
  - 它支持语音消息、表情包识别、图片识别

它运行在你的电脑上，通过QQ和你聊天。


──────────────────────────────────────────────────────────────
  6.2 你需要准备什么？
──────────────────────────────────────────────────────────────

1. 一台Windows电脑（Win10/Win11都行）
2. 能上网
3. 一个QQ号（用来让AI通过QQ和你聊天）
4. 大约10-15分钟的耐心

不需要你懂编程！跟着下面的步骤一步步来就行。


──────────────────────────────────────────────────────────────
  6.3 详细安装步骤
──────────────────────────────────────────────────────────────

【步骤1：运行一键部署】
  在解压出来的文件夹里，找到"一键部署.bat"，双击它。
  整个过程可能需要3-10分钟，请耐心等待。
  看到"部署完成！"就说明成功了。

【步骤2：配置API密钥】
  双击"启动CROWN配置终端.bat"，会打开一个网页界面。
  这是你的"控制台"，所有配置都在这里改。

  最重要的一步：配置API密钥（详见上方"三、API配置详解"）
    1. 点击左侧的"模型配置"
    2. 选择你使用的API平台
    3. 粘贴你的API密钥
    4. 选择模型
    5. 保存

【步骤3：配置QQ登录】
  启动机器人后会弹出NapCat窗口。
  第一次使用需要扫码登录：
    1. 窗口里会显示一个二维码
    2. 用手机QQ扫码登录
    3. 登录成功后窗口会显示连接状态

【步骤4：启动机器人】
  双击"重启文明（启动聊天ai）.bat"。
  看到"启动成功"之类的提示就OK了。

【步骤5：开始聊天！】
  用你的另一个QQ号给机器人QQ发消息。
  机器人会自动回复你。


──────────────────────────────────────────────────────────────
  6.4 怎么和AI聊天？
──────────────────────────────────────────────────────────────

直接发消息就行！就像和朋友聊天一样。

常用命令：
  /clear        清空聊天记录，重新开始
  /memory       查看AI记住了哪些关于你的事
  /profile      查看你们的关系状态和好感度
  /debug        查看调试信息
  /恋人模式 开   开启恋人模式
  /恋人模式 关   关闭恋人模式
  /relationship  查看/切换关系类型


──────────────────────────────────────────────────────────────
  6.5 关系系统
──────────────────────────────────────────────────────────────

机器人支持多种关系类型：
  - 朋友（默认）
  - 恋人（lover）
  - 闺蜜（bestie）
  - 师徒（mentor）
  - 夫妻（spouse）
  - 家人（family）
  - 宠物（pet）

切换关系：
  发送 /relationship 查看当前状态
  发送 /relationship lover 切换到恋人模式


──────────────────────────────────────────────────────────────
  6.6 语音功能
──────────────────────────────────────────────────────────────

机器人支持语音回复（需要配置TTS）。

配置方法：
  1. 在配置终端的"模型配置"中找到TTS设置
  2. 填入TTS API的地址和密钥
  3. 准备一个参考音频文件（角色的声音样本）
  4. 把参考音频放到 voice/ 文件夹里
  5. 在配置中指定参考音频路径

默认使用小米MiMo的TTS服务（音色克隆功能）。


──────────────────────────────────────────────────────────────
  6.7 联网搜索
──────────────────────────────────────────────────────────────

机器人可以联网搜索来回答你的问题。

配置方法：
  1. 注册一个 Tavily API 账号（tavily.com）
  2. 获取API Key
  3. 在配置终端的"模型配置"中填入搜索API密钥

使用方式：
  直接问需要联网搜索的问题，AI会自动判断是否需要搜索。


──────────────────────────────────────────────────────────────
  6.8 主动消息（碎碎念）
──────────────────────────────────────────────────────────────

机器人会主动找你聊天。

配置方法（在配置终端的"主动消息"页面）：
  - interval_hours：主动消息间隔（默认0.5小时）
  - quiet_hours_start/end：深夜静默时段（默认0-7点）
  - mutter_enabled：是否开启碎碎念
  - care_trigger_chance：主动关心触发概率


──────────────────────────────────────────────────────────────
  6.9 记忆系统
──────────────────────────────────────────────────────────────

机器人有四层记忆：
  1. 工作记忆：最近的对话（滑动窗口）
  2. 短期记忆：最近记住的事情
  3. 长期记忆：重要的回忆（会随时间衰减）
  4. 深层记忆：模糊的印象（夜间回味发酵）

查看记忆：发送 /memory
让AI遗忘：发送 /forget 关键词


══════════════════════════════════════════════════════════════
  七、常见问题
══════════════════════════════════════════════════════════════

Q: 双击"一键部署.bat"后闪退了怎么办？
A: 右键点击它，选"以管理员身份运行"。

Q: 启动后AI不回复怎么办？
A: 检查以下几点：
   1. config.yaml 里的 api_key 填了没有？
   2. 网络能正常上网吗？
   3. QQ登录成功了吗？（看NapCat窗口）
   4. 你是否在白名单中？
   5. 等10秒再试，第一次回复比较慢

Q: 提示"API错误"怎么办？
A: 检查密钥有没有复制完整、有没有过期、余额够不够。

Q: AI回复的内容很奇怪/重复怎么办？
A: 发 /clear 清空聊天记录，重新开始聊。

Q: 怎么更换AI的性格？
A: 在配置终端的"人设控制"页面修改，或者编辑 personas/ 下的YAML文件。

Q: 怎么让AI只和特定的人聊天？
A: 使用白名单功能。打开配置终端 -> 白名单 -> 添加允许的QQ号。

Q: 数据库在哪里？
A: 新版在 data/accounts/{QQ号}/{人设}/user_data.db。
   可以在配置终端的"数据库"页面查看和管理。

Q: 怎么备份我的配置？
A: 在配置终端的"备份管理"页面，点击"创建备份"。

Q: 怎么更新到新版本？
A: 下载新版本后，把你的 config.yaml、personas/、data/、voice/
   复制到新版本目录。首次启动会自动迁移数据库结构。

Q: debug.log 在哪里？
A: 在项目根目录的 debug.log 文件。

【陪伴中枢】

陪伴中枢位于 WebUI 左侧导航栏，用来把记忆、成长目标、主动消息、安全状态、心理画像、关系绑定和 AI 配置集中显示在一个页面里。

使用方法：
1. 双击“启动CROWN配置终端.bat”打开 WebUI。
2. 点击左侧“陪伴中枢”。
3. 如果只想看某个用户或某个人设，在顶部输入用户 ID 或人设名称；不填则显示当前能读到的总体摘要。
4. 查看“主动消息判断”区域，可以看到最近主动消息的 sent、skipped、failed、planned 状态，以及触发原因或跳过原因。
5. 每个卡片右上角有跳转按钮，可进入对应的详情页继续查看。

陪伴中枢是只读页面，不会直接改数据库。CROWN 自用版不安装、不启用“过度依赖”监控模块。

【系统自检】

系统自检位于 WebUI 左侧导航栏，用来确认项目结构、聊天模块、WebUI 页面和数据库表是否真实接上。

使用方法：
1. 双击“启动CROWN配置终端.bat”打开 WebUI。
2. 点击左侧“系统自检”。
3. 点击“刷新自检”。
4. 查看“聊天与数据库模块”，确认聊天入口、记忆账本、成长目标、主动消息、安全护栏、关系定制、账号绑定和心理画像的文件是否存在，是否已经接到对应数据库表。
5. 查看“WebUI 页面联通性”，确认每个页面是否有对应 API，是否作用在文件、运行时状态或数据库上。
6. 查看“数据库与表”，确认当前项目实际发现了哪些数据库文件和表。

状态含义：
  可用       文件、接口和已需数据表都已接上。
  部分可用   主链路存在，但部分表还没出现或仍需聊天触发生成。
  等待数据   代码和接口存在，当前数据库尚未产生对应表。
  接口缺失   页面定义的接口没有注册，需要修复。
  文件缺失   模块文件不存在，需要补齐或调整路径。

系统自检只读，不会写入或删除聊天记录、记忆、目标、关系或配置。

【内部上下文构建】

聊天主流程现在由 `src/core/pipeline.py` 负责调度，由 `src/core/context_builder.py` 负责拼装回复前上下文。

它会把以下内容集中送入聊天提示词：
  时间与生活状态
  长期记忆、情景记忆、记忆账本
  情绪、PAD、心理画像、人格控制
  关系定制、账号绑定、成长目标
  内容策略、用户数据摘要、额外上下文

因此 WebUI 中修改的关系、账号绑定、成长目标、记忆和心理画像会真实参与聊天，而不是只保存在页面上。

【安全协调器】

聊天安全入口现在由 `src/core/safety_coordinator.py` 统一协调。

它负责：
  接收消息后的危机等级和边界风险评估
  高危输入的直接安全回复
  回复前的安全协议提示注入
  回复后的输出护栏
  主动消息发送前的安全前置检查

底层仍使用 `src/safety/safety_monitor.py`。CROWN 自用版不安装、不启用“过度依赖”监控模块。

══════════════════════════════════════════════════════════════
【记忆协调器】

聊天记忆写入现在由 `src/core/memory_coordinator.py` 统一协调。

它负责：
  用户输入的长期记忆抽取
  明确事实、偏好、目标和风险写入 MemoryLedger
  AI 回复写入短期记忆
  每隔一定轮数触发长期记忆 AI 抽取
  按事件类型和情绪强度写入情景记忆

如果出现“刚刚说过又问”的情况，优先检查系统自检中的记忆模块、数据库页里的 `chat_history` / `memory_ledger` / `long_term_memory`，再看 debug.log 中的 memory 相关日志。

══════════════════════════════════════════════════════════════
【成长协调器】

聊天成长状态现在由 `src/core/growth_coordinator.py` 统一协调。

它负责：
  回复完成后更新关系成长统计
  从“计划、目标、打算、想完成”等聊天内容中创建成长目标
  每隔一定轮数触发成长总结
  把对话分类成日常分享、分享秘密、关心回应、情感共鸣或日常问候

这些记录会进入成长系统和进化日志，并被 WebUI 的成长目标页、陪伴中枢和后续聊天上下文读取。想确认是否生效，可以先看系统自检中的成长目标模块，再看成长目标页和陪伴中枢里的目标、关系成长与最近事件。

══════════════════════════════════════════════════════════════
【工具路由协调器】

聊天里的工具路由现在由 `src/core/tool_router.py` 统一协调。

它负责：
  在回复前注入工具能力提示（工具可用时）
  检测模型回复里是否包含工具调用信号
  工具调用成功后回注结果并生成二次回复
  二次回复继续经过安全护栏

如果想确认这一块是否生效，可以在系统自检里看聊天入口模块是否包含 `src/core/tool_router.py`，并在 debug.log 中查看工具调用相关日志。

══════════════════════════════════════════════════════════════
【评审协调器】

回复后的人设稳定评审现在由 `src/core/critic_coordinator.py` 统一协调。

它负责：
  回复前注入人格漂移修正提示（存在时）
  回复后缓存文本用于漂移检测
  异步触发人格漂移检查
  计算残差并执行可采纳性判定，必要时触发修正信号

如果想确认是否生效，可以在系统自检里看聊天入口模块是否包含 `src/core/critic_coordinator.py`，并在 debug.log 中查看 persona 相关日志。

【聊天记录导入模块】
作用：把历史聊天（文本或文件）导入到聊天记录库，供关系简史、记忆冷启动和分析模块读取。
使用：打开 WebUI → 左侧「聊天记录导入」→ 输入 user_id（必填）→ 选择粘贴导入或文件导入。
支持文件：zip/html/htm/txt/mht。
注意：该模块会写入 chat_records 与 chat_record_analysis，但不会直接改写记忆账本、关系状态和成长目标；CROWN 自用版不接入“过度依赖”监控。

【关系简史模块】
作用：在「关系定制」页集中展示关系切换、账号绑定、成长事件、关系相关记忆、聊天记录和长期记忆，帮助确认当前关系判断依据。
使用：打开 WebUI → 左侧「关系定制」→ 在“关系简史（只读）”填写 user_id（建议）与 persona（可选）→ 点击“刷新简史”。
可查看：当前关系、最近切换时间、绑定关系、最近联系时间，以及按时间排序的事件流。
注意：该模块只读，不直接修改数据库；若某些表尚未生成会显示空态；CROWN 自用版不接入“过度依赖”监控。

【长期记忆冷启动摘要】
作用：从已导入聊天记录中提炼初始记忆候选，写入 memory_ledger 与 long_term_memory，降低新账号“无上下文”状态。
使用：打开 WebUI → 左侧「聊天记录导入」→ 确认 user_id（可选 persona）→ 点击“生成冷启动摘要”。
可查看：样本数、关键词、候选摘要内容、memory_ledger 与 long_term_memory 入账数量。
注意：该模块会写库，请在记忆账本页继续确认和整理；CROWN 自用版不接入“过度依赖”监控。

【评测控制台】
作用：把聊天主链路相关能力做只读评估，快速判断“缺数据”还是“链路故障”。
使用：打开 WebUI → 左侧「评测控制台」→（可选）填写 user_id 与 persona → 勾选场景 → 点击“运行评测”。
历史：可按状态（通过/警告/失败）和 report_id 关键词筛选历史报告并查看详情。
注意：该页面只读，不直接修改聊天数据和配置；waiting_db/空态通常表示该账号尚未触发该能力。

【高级配置自动归类】
作用：WebUI 会把复杂模块自动归入「高级设置」，常用配置区保持简洁。
当前归类示例：系统自检、评测控制台、记忆账本、聊天记录导入、数据库等。
注意：自动归类只影响导航展示，不影响接口行为和数据库内容。
